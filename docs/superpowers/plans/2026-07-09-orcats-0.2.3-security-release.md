# Orcats 0.2.3 Security Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish `@twelvehart/orcats@0.2.3` with no known dependency advisories, a working npm CLI entrypoint, and unchanged runtime behavior.

**Architecture:** Restore the package's tracked Bun shim, then upgrade only the ESLint boundaries plugin that owns the vulnerable Handlebars chain. Migrate its v5 rule to v6 object selectors while explicitly retaining import-only analysis, then align all release metadata and publish through the existing tag-driven Trusted Publishing workflow.

**Tech Stack:** Bun 1.3, TypeScript 5.9, ESLint 9, `eslint-plugin-boundaries` 6.0.2, npm Trusted Publishing, GitHub Actions.

## Global Constraints

- Target package and tag are exactly `@twelvehart/orcats@0.2.3` and `v0.2.3`.
- Upgrade `eslint-plugin-boundaries` to `^6.0.2`; do not add an override or upgrade to v7.
- Preserve v5 behavior with `"boundaries/dependency-nodes": ["import"]`.
- Preserve the existing model and conversation allow/disallow policy exactly.
- Restore tracked `bin/orcats`; do not include the main checkout's self-dependency or `package-lock.json`.
- Runtime dependencies remain exactly `effect`, `neverthrow`, and `zod`.
- Publish only through `.github/workflows/release.yml`; do not run local `npm publish` or add `NPM_TOKEN`.
- Merge only after GitHub CI passes and complete an adversarial review before final validation.

```javascript
settings: {
  "boundaries/dependency-nodes": ["import"],
  "boundaries/elements": [
    { "type": "model", "pattern": "src/model/*" },
    { "type": "conversation", "pattern": "src/conversation/*" },
    { "type": "flow", "pattern": "src/flow/*" },
    { "type": "tools", "pattern": "src/tools/*" },
    { "type": "backends", "pattern": "src/backends/*" },
    { "type": "runner", "pattern": "src/runner/*" },
    { "type": "cli", "pattern": "src/cli/*" },
    { "type": "testing", "pattern": "src/test-utils/*" }
  ]
},
rules: {
  "boundaries/dependencies": [
    "error",
    {
      "default": "allow",
      "rules": [
        {
          "from": { "type": "model" },
          "disallow": [
            {
              "to": {
                "type": [
                  "conversation",
                  "flow",
                  "tools",
                  "backends",
                  "runner",
                  "cli",
                  "testing"
                ]
              }
            }
          ]
        },
        {
          "from": { "type": "conversation" },
          "disallow": [
            {
              "to": {
                "type": ["flow", "tools", "backends", "runner", "cli"]
              }
            }
          ]
        }
      ]
    }
  ]
}
```
